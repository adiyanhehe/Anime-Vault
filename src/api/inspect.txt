They have ZERO infrastructure — every "download link" just redirects to a Google Drive direct download URL. We will BUILD a complete replacement:
- A scraper that extracts all their GDrive file IDs
- A transcoding pipeline that converts GDrive files into HLS (.m3u8) streams
- A clean API that serves stream URLs with audio track selection and subtitle selection
- A web player with zero ads, zero gambling, zero popups
- TMDB metadata for proper search and discovery

### 2.4 Download Link Extraction (THE CRITICAL PART)
Each download button links to:
https://{mirror_subdomain}.movielinkbd.li/movie/{another_slug}

These {slug} values need to be decoded to extract the underlying Google Drive FILE ID.

DECODING METHODS TO TRY:

Method 1: Base64 Decode
- Try: atob(slug) → may be standard base64
- Try: atob(slug.replace(/-/g,'+').replace(/_/g,'/')) → URL-safe base64
- If successful, the result may be a GDrive file ID (28+ chars, alphanumeric + underscore + hyphen)

Method 2: If decoding fails, the slug itself IS the identifier
- The actual GDrive file ID may be behind the Cloudflare redirect
- Follow the redirect chain (using requests with allow_redirects=True, but Cloudflare blocks this)
- Alternative: use the Wayback Machine's cached version of the download page

Method 3: Direct pattern matching
GDrive file IDs look like: 0ByeS4oOUV-49Zzh4R1J6R09zazQ or 1LDRlyZaZTF-bxHjBDEiL2obMxYHtf3YT
Pattern: [a-zA-Z0-9_-]{28,}
Search the page HTML for this pattern and extract all matches

Once you have the FILE_ID:
Direct download URL: https://drive.google.com/uc?export=download&id={FILE_ID}
Stream/preview URL:  https://drive.google.com/file/d/{FILE_ID}/preview
Video info API:       https://drive.google.com/get_video_info?docid={FILE_ID}

## SECTION 4: GOOGLE DRIVE → HLS TRANSCODING PIPELINE

### 4.1 GDrive Video Probing
Use yt-dlp's Python API to probe the GDrive file:

```python
import yt_dlp

def probe_gdrive_video(file_id):
    url = f"https://drive.google.com/uc?export=download&id={file_id}"
    ydl_opts = {
        'quiet': True,
        'no_warnings': True,
        'listformats': True,  # Just list, don't download
    }
    with yt_dlp.YoutubeDL(ydl_opts) as ydl:
        info = ydl.extract_info(url, download=False)
    return info
This returns a dict containing:

formats: list of video/audio/subtitle formats
subtitles: dict of subtitle tracks by language
automatic_captions: auto-generated captions by language
duration, title, thumbnail

def extract_audio_track(file_id, audio_format_id, output_path):
    ydl_opts = {
        'format': audio_format_id,
        'outtmpl': output_path,
        'quiet': True,
    }
    url = f"https://drive.google.com/uc?export=download&id={file_id}"
    with yt_dlp.YoutubeDL(ydl_opts) as ydl:
        ydl.download([url])
4.3 Subtitle Extraction
Probe for embedded subtitles using yt-dlp:

python



def extract_subtitles(file_id):
    url = f"https://drive.google.com/uc?export=download&id={file_id}"
    ydl_opts = {
        'quiet': True,
        'writesubtitles': True,
        'writeautomaticsub': True,
        'subtitleslangs': ['all'],
        'skip_download': True,
        'outtmpl': f'subs/{file_id}/%(id)s.%(ext)s',
    }
    with yt_dlp.YoutubeDL(ydl_opts) as ydl:
        info = ydl.extract_info(url, download=True)
    
    # info['subtitles'] will contain:
    # {
    #   'en': [{'url': 'https://...', 'ext': 'vtt'}],
    #   'hi': [{'url': 'https://...', 'ext': 'vtt'}],
    #   'es': [{'url': 'https://...', 'ext': 'vtt'}],
    #   ...
    # }
    return info.get('subtitles', {})
Fallback if no embedded subs: Use OpenSubtitles API

python



def fetch_opensubtitles(imdb_id, languages=['en', 'hi', 'es', 'ar', 'id', 'th', 'zh', 'ms', 'vi']):
    headers = {
        'Api-Key': 'YOUR_OS_API_KEY',
        'Content-Type': 'application/json',
        'User-Agent': 'YourApp v1.0'
    }
    # Search subtitles by IMDB ID
    params = {
        'imdb_id': imdb_id,
        'languages': ','.join(languages),
        'type': 'movie'  # or 'episode'
    }
    resp = requests.get('https://api.opensubtitles.com/api/v1/subtitles', 
                       headers=headers, params=params)
    data = resp.json()
    
    subtitles = {}
    for sub in data.get('data', []):
        lang = sub['attributes']['language']
        if lang not in subtitles:
            # Download the subtitle file
            dl_resp = requests.post(
                'https://api.opensubtitles.com/api/v1/download',
                headers=headers,
                json={'file_id': sub['id']}
            )
            sub_url = dl_resp.json().get('link')
            subtitles[lang] = [{'url': sub_url, 'ext': 'vtt'}]
    
    return subtitles

4.4 HLS Transcoding with FFmpeg
For each video quality, create separate HLS streams:

bash



# Single command: split video + extract audio tracks + convert subs

# Step 1: Download the GDrive video stream (piped, not saved to disk)
# Step 2: Transcode to HLS with all audio tracks as separate renditions

ffmpeg -i "https://drive.google.com/uc?export=download&id=FILE_ID" \
  -map 0:v:0 -c:v libx264 -preset fast -crf 23 \
    -b:v:0 5000k -maxrate:v:0 7500k -bufsize:v:0 10000k \
    -filter:v:0 "scale=-2:1080" \
  -map 0:v:0 -c:v libx264 -preset fast -crf 23 \
    -b:v:1 2500k -maxrate:v:1 3750k -bufsize:v:1 5000k \
    -filter:v:1 "scale=-2:720" \
  -map 0:v:0 -c:v libx264 -preset fast -crf 23 \
    -b:v:2 1000k -maxrate:v:2 1500k -bufsize:v:2 2000k \
    -filter:v:2 "scale=-2:480" \
  -map 0:a:m:language:kor -c:a:0 aac -b:a:0 128k \
    -var_stream_map "v:0,a:0 v:1,a:0 v:2,a:0" \
  -f hls -hls_time 10 -hls_list_size 0 \
    -hls_segment_filename "streams/%v/segment_%03d.ts" \
    -master_pl_name "master.m3u8" \
    "streams/%v/index.m3u8"

SECTION 9: CACHING STRATEGY
Redis Cache Keys:



scrape:homepage:{mirror_domain} → HTML of homepage (TTL: 1 hour)
scrape:detail:{slug_id} → HTML of detail page (TTL: 6 hours)
gdrive:probe:{file_id} → yt-dlp probe result (TTL: 24 hours)
gdrive:subtitles:{file_id} → subtitle dict (TTL: 48 hours)
tmdb:search:{query}:{type} → TMDB search results (TTL: 7 days)
tmdb:details:{tmdb_id} → TMDB item details (TTL: 7 days)
hls:manifest:{content_id}:{episode}:{quality}:{audio} → Master playlist (TTL: 1 hour)
hls:segment:{segment_id} → TS segment (TTL: 1 hour, or until purged)
GDrive Quota Protection:
Track daily GDrive requests per file ID
Max 10 probes per file ID per day (GDrive has 750GB/day limit)
Cache probe results aggressively
Use exponential backoff on 403 responses
SECTION 10: ERROR HANDLING


Scenario	Handling
Cloudflare blocks scraper	Rotate to another mirror domain, wait 30s, retry with different user agent
GDrive file returns 403 (quota)	Return cached stream if available, otherwise return "temporarily unavailable"
No embedded subtitles	Fall back to OpenSubtitles API
No OpenSubtitles results	Return available subtitles only, mark missing languages
FFmpeg transcoding fails	Fall back to direct GDrive URL (via video tag with Google's direct mp4)
All mirrors dead	Return empty results with status "source_unavailable"
TMDB API fails	Return partial data with available metadata from movielinkbd page
Memory pressure during transcoding	Limit concurrent FFmpeg processes to CPU core count

Key Design Decisions:
yt-dlp probes GDrive directly — no need to download the full file to list formats
FFmpeg transcodes on-the-fly — segments are cached, not regenerated
OpenSubtitles as fallback — covers 90%+ of missing embedded subs
TMDB for metadata — proper posters, overviews, genres (not scraped from movielinkbd)
Audio tracks as separate HLS playlists — player switches audio without buffering video
Subtitles as static VTT files — served from disk/cache, minimal overhead
Mirror rotation — if one domain is blocked, auto-switch to another
Preferences persisted — user's audio/subtitle choices remembered across sessions
